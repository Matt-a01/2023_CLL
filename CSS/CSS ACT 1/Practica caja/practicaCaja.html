Modelo de caja

Descripcion

la especificacion de CSS asume que el documento se modela como un arbol de elementos . cada elemento contiene una lista ordenada de cero o mas elementos secundarios con una cadena opcional de texto antes de la lista, enmedio de los hijos y despues de la lista. El unicp elemento que no tiene padre es el elemento raiz

CSS descripe como deben de posicionarse cada elemento y cada cadena de tecto, tranformado un arbo de documentos en un arbol de cajas rectángulares, cuyotamaño y/o posiciones depende de sus propiedades. Existen Cajas a nivel de bloque, Cajas a nivel de línea y Cajas a nivel interno (inline. Una caja a nivel de bloqule es como un párrafo, una caja a nivel de línea es como una línea de texto y las cajas a nivel de interno son como las palabras de una línea

las teres cajas se relaciuonean estrechamente con el arbol de elementos. canda elemento se trasforma en cero o mas cajas, si el elemento no es la raíz, entonses cada una de las cajas es un hijo de una caja del elemento padre o un hijo de la caja del propio elemento. En otras palabras puede haber mas menos elementos, pero cada caja pertenence a un elemento.

las propiedades se establecen sobre los elementos e influyen en la forma en que cadenaelemento se convierte en cajas, pero en esta especieficacion hay una intercambiabilidad de la propiedad P de un elemento y la propiedad P de una caja a menos que se a importante distinguir entre la caja y el elemento, por ejemplo, cuando el elemento tiene carias cajas y no todas elleas tienen el el mismo valor para la propiedad.

Las cajas tienen relleno, un borde y margenes (ver la figura). diferentes propiedades determinan el ancho (que podrian ser cero) de cada uno de ellos.

Cada caja de tiene un área de contenido. El rectangulo que rodea esta área es la orilla de contenido. Alrededor del área de contenido del área de relleno sus bordes exteriores son llamados orilla del relleno. por fuera del relleno esta el área de borde y el límite exterior de exterior de esta área es la orilla del borde. Finalmente, por e fuera del borde ese escuentre el orilla del margen.

Cueando la especificicación dice que el borde de relleno esta ausente, en algun lado de la caja, en realidad significa que su ancho es cero.

una caja es horizontal si su propiedad blok-progretion tiene valor de tb, en caso contriario es vertical.

Tipos de cajos

La propiedad en conjunto con floatpostition determina el tipo de caja que se generan para un elemento. Los valores son los que siguen:

inline
Cajas de lineas
block
Cajas de bloque
inline-block
Cajas de que se seguia por una sola caja en linea, de manera similar a un elemento reemplazado. El interior de un bloque en linea se trasforma como euna caja negra, y la caja por si mismo es formateada como una caja en linea
List-item
Una o mas cajas de bloque y una o mas cajas de marcado. las cajas de mercado se definen en las listas de modulo.
run-in
Cajas de bloque o lunea dependiendo el contexto. Las porpiedades se aplican a cajas de ejecucion interna (run-in) en base a su estado final.
